2. Detailed Agent Specifications & Core Responsibilities

🤖 1. Call Optimization & Router Agent

Model Configuration: gpt-4o-mini (Optimized for ultra-low latency, token economy, and deterministic intent classification).

A. Core Action Items & Execution Steps

  • Intent Classification: Parses incoming message payloads (text or Whisper-transcribed audio) to classify user intent into one of three distinct tracks: [ADMIN_FAQ, MEDICAL_INTAKE, CRITICAL_EMERGENCY].
  • IVR-to-WhatsApp Deflection: Triggers an API webhook to telecom providers when voice channel queues exceed SLA limits, gracefully transferring the user session to an asynchronous text environment.
  • Dynamic Guardrail Scanning: Intercepts conversational inputs to match against an explicit medical "Red-Flag Dictionary" concurrently during processing.

B. Edge Cases & Exception Handling Matrix

Edge Case Encountered Agent Resolution Strategy
Ambiguous User Input (e.g., "I feel bad today" without specific symptoms). Routes to Intake Agent with an ambiguity flag, forcing the intake logic to ask a broad open-ended clarification question.
Multi-Intent Inputs (e.g., "I want to ask about insurance AND I have a bad headache"). Splits the state. Immediate execution of Emergency Bypass Check on the medical symptom, postponing the FAQ pipeline.

🩺 2. Patient History & Smart Intake Agent

Model Configuration: Claude 3.5 Sonnet (Leveraged for deep contextual medical reasoning, entity extraction, and multi-modal clinical pattern matching).

A. Core Action Items & Execution Steps

  • Multi-Modal Ingestion & OCR: Processes raw images of handwritten prescriptions, medical reports, and lab results via Vision LLM layers. Converts unstructured text to standard chemical/generic nomenclature.
  • GraphRAG Medical Profiling: Queries the Neo4j Graph DB using patient identifiers to look up structural family connections, extracting direct genetic risks (e.g., immediate family history of cardiovascular disorders or diabetes).
  • SOAP Note Synthesis: Organizes medical inputs into a structured clinical document following the standard Subjective, Objective, Assessment, and Plan (SOAP) medical reporting blueprint.

B. Edge Cases & Exception Handling Matrix

Edge Case Encountered Agent Resolution Strategy
Illegible or corrupted prescription/document image upload. Applies specialized image preprocessing. If OCR confidence drops below 75%, it initiates a polite conversational request: "I couldn't quite read that document clearly. Could you take a clearer photo or type the medication name?"
Conflicting information (e.g., Patient states they don't take medications, but the Vision OCR identifies an active prescription). Flags a specific Clinical_Discrepancy notice highlighted in red directly inside the generated SOAP Note for the reviewing doctor.

🚨 3. Emergency Bypass Guardrail (Safety Engine)

Model Configuration: Hard-coded deterministic rule matrices paired with a fine-tuned binary classifier model (Zero-latency processing pipeline).

A. Core Action Items & Execution Steps

  • Immediate State Interruption: Forces a complete system-wide override of any active LangGraph loops the moment high-risk indicators (e.g., crushing chest pain, sudden dyspnea, severe anaphylaxis) are identified.
  • Automated Dispatch Core: Fires location-gathering APIs to extract the user's instantaneous GPS/cellular coordinates and automatically populates emergency ambulance dispatch schemas.
  • Human-in-the-Loop (HITL) Forced Escalation: Opens a low-latency WebSocket stream that pushes a visual flashing alarm accompanied by an audible warning tone directly onto the ER Nurse Console Dashboard.

B. Edge Cases & Exception Handling Matrix

Edge Case Encountered Agent Resolution Strategy
The system drops the real-time WebSocket connection to the physical ER Nurse dashboard mid-crisis. Executes an immediate automated fallback loop: initiates an urgent SMS backup transmission and triggers an outbound telephony synthetic voice call to the hospital
